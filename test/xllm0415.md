### 2. Transformer 核心 Attention 计算优化替换
传统的 Attention 实现由于 `k_cache/v_cache` 的切分可能会大量引入诸如 `.to(kCPU)` 或者冗长的标量数据传递影响显存带宽。

在预发（Pre-fill）阶段和解码（Decoder/Paged Attention）阶段的代码 [xllm/core/layers/npu_torch/attention.cpp](../xllm/core/layers/npu_torch/attention.cpp) 处，我植入了使用定制 `x_flash_attention_infer` 与 `x_paged_attention` 等新算子的接口位点：

```cpp
    // 使得 Attention Score 计算不仅省去 .to(CPU) 还缩减 Block Table 查询访存
    // xllm::kernel::neo::x_custom_paged_attention_acl_graph(...)
  ...
#ifdef USE_NEO_FUSED_OPS
    // PagedAttention 的标准 NPU 算子调用切换：
    // xllm::kernel::neo::x_paged_attention(query, k_cache, ...);
```

### 总结我们现已实现的融合提优靶点 (使用 `USE_NEO_FUSED_OPS=1` 测试)

1. **RMS Norm & GATED Linear Projection**: `qwen3_gated_delta_net_base.cpp` (针对 `Sub`、`Mul` 微小碎片循环消除)、 `dense_mlp.cpp`

### 大融合替换方案的等价性分析

#### 1. Chunk GDN Attention 矩阵构建 ([qwen3_gated_delta_net_base.cpp:175](../xllm/core/layers/npu_torch/qwen3_gated_delta_net_base.cpp#L175))

- **原始**: 双层嵌套循环 `for i in 1..64: Slice → Mul → Sum → IndexPut` = ~2000 次微算子/chunk
- **替换**: Neumann 级数 `L = L + L@L` 迭代 6 次 (`log2(64)=6` 次 MatMul)
- **等价性**: `(I-L)^{-1}` 的 Neumann 级数在严格下三角矩阵 L 上有限步精确收敛
- **预期收益**: 消除 280K 次 Slice (10%) 中 chunk-internal 部分

#### 2. Decode GDN 隐状态递推 ([qwen3_gated_delta_net_base.cpp:515](../xllm/core/layers/npu_torch/qwen3_gated_delta_net_base.cpp#L515))

- **原始**: `torch_recurrent_gated_delta_rule` = `for(i in seq_len): Select → BMM → Mul → Exp → Add`
- **替换**: `fused_recurrent_gated_delta_rule` (triton_npu 自定义 Kernel)
- **等价性**: 输入输出完全相同 (q,k,v,g,beta,initial_state → output,final_state)，Kernel 内部在 NPU UB 中保持 state
- **预期收益**: 消除 decode 阶段所有 Select/Mul/BMM 碎片

#### 3. Flash Attention Prefill ([attention.cpp:97](../xllm/core/layers/npu_torch/attention.cpp#L97))

- **原始**: `atb::npu_flash_attention` → ATB SelfAttention(PA_ENCODER)
- **替换**: `neo_batch_prefill` → `x_flash_attention_infer` (Catlass 框架, BlockMmadQK + OnlineSoftmax + BlockMmadPV)
- **等价性**: 都计算 `softmax(Q·K^T/√d)·V`，数学结果一致
- **预期收益**: 优化 UnpadFlashAttentionBF16NdKernel (4.7s, 16.4%)

#### 4. FusedAddRmsNorm ([qwen3_next_hybrid_decoder_layer_base.cpp:128](../xllm/core/layers/npu_torch/qwen3_next_hybrid_decoder_layer_base.cpp#L128))

- **原始**: `x.to(FP32) + residual.to(FP32)` → `RmsNorm` = 4 步 (2x Cast + Add + Norm)
- **替换**: `npu::add_rms_norm(x, residual, gamma, eps)` = 1 步
- **等价性**: `output = RmsNorm(x + residual, gamma, eps)` 数学完全一致
- **参考**: `ops-nn/norm/add_rms_norm` AscendC 实现, ATB `RMS_NORM_PRENORM` 模式
- **预期收益**: 消除 BF16→FP32 Cast + 中间 FP32 张量 HBM 读写

#### 5. MoE GroupedMatmul + SwiGLU 融合 ([fused_moe.cpp:332](../xllm/core/layers/npu_torch/fused_moe.cpp#L332))

- **原始**: `GroupedMatmul → Activation(SiLU+Mul) → GroupedMatmul` = 3 次 Kernel
- **替换**: `GroupedMatmulV4(act_type=3)` = 2 次 Kernel (gemm1+act 融合)
- **等价性**: SwiGLU(x) = SiLU(x[gate]) * x[up]，`act_type=3` 实现相同计算
- **参考**: `xllm_ops_neo/moe_grouped_matmul_swiglu_quant` (AIC+AIV 并行)
- **预期收益**: 消除 gemm1→activation 间的 HBM 中间张量读写

#### 6. DenseMLP SwiGLU 融合 ([dense_mlp.cpp:100](../xllm/core/layers/common/dense_mlp.cpp#L100))

- **原始**: `gate_up → act_->forward(gate_up, output)` (可能拆分为 SiLU + Mul)
- **替换**: `npu::active(gate_up, "silu")` 使用 CANN aclnnActivation 单 Kernel SwiGLU
- **等价性**: `SiLU(gate) * up` 完全一致
- **预期收益**: 减少 DenseMLP 层的 Mul 碎片算子
